# Telegram Bot & Admin Panel (Monorepo)

Добро пожаловать в проект! Это монорепозиторий, объединяющий Telegram-бота, вспомогательный REST API-сервер на FastAPI, а также кроссплатформенную панель администратора на **Tauri 2 + React + TypeScript** (для ПК и Android).

---

## 📂 Структура репозитория

*   **`bot/`** — Основной Telegram-бот на Python (использует `aiogram` и `sqlite3` в WAL-режиме).
*   **`bot-api/`** — REST API сервер на `FastAPI`, обеспечивающий доступ к базе данных бота для панелей управления.
*   **`admin-panel/`** — Десктопная версия панели управления администратора (Tauri v2 + React + Vite).
*   **`admin-panel-android/`** — Мобильная версия панели управления администратора для Android (Tauri v2 Android).
*   **`start_api.sh`** — Скрипт для быстрого ручного запуска API сервера.

---

## ⚙️ 1. Настройка и запуск Telegram-бота (`bot/`)

Бот является основным ядром системы и управляет играми, балансом и модерацией пользователей.

### Установка зависимостей:
1. Перейдите в папку бота:
   ```bash
   cd bot
   ```
2. Создайте и активируйте виртуальное окружение Python:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

### Настройка:
Скопируйте `config.example.json` в `config.json`:
```bash
cp config.example.json config.json
```
Отредактируйте `config.json` и укажите параметры:
*   `bot_token`: Токен вашего Telegram-бота, полученный от `@BotFather`.
*   `owner_id`: Ваш Telegram ID (главный владелец).
*   `admins`: Список ID главных администраторов.

### Запуск:
```bash
python bot.py
```
При первом запуске бот автоматически создаст базу данных `game.db` и применит необходимые миграции.

## 🌐 2. REST API Сервер (`bot-api/`)

API обеспечивает связь между панелями администратора и общей базой данных `game.db`.

### Установка:
Использует то же виртуальное окружение, что и бот.
```bash
cd bot-api
# Убедитесь, что виртуальное окружение в корне активировано
```

### Запуск:
Вы можете запустить API-сервер вручную:
```bash
# Из корневой директории:
./start_api.sh
```
Или напрямую через uvicorn:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```
*Примечание:* Для работы мобильной панели API-сервер запускается на хосте `0.0.0.0`, чтобы телефон мог подключиться к компьютеру по Wi-Fi.

## 💻 3. Панель управления ПК (`admin-panel/`)

Десктопное приложение для управления ботом (пользователи, комнаты, трансляции, настройки экономики).

### Установка:
1. Установите зависимости Node.js:
   ```bash
   cd admin-panel
   npm install
   ```
2. Убедитесь, что у вас установлены системные зависимости Rust/Tauri (Cargo, rustc). Подробнее в [руководстве Tauri](https://v2.tauri.app/start/prerequisites/).

### Запуск в режиме разработки:
При запуске панели Tauri автоматически проверит доступность порта `8000` и запустит API-сервер, если он ещё не запущен.
```bash
npm run tauri dev
```

### Сборка готового приложения:
```bash
npm run tauri build
```

## 📱 4. Мобильная панель Android (`admin-panel-android/`)

Специальная версия панели администратора, адаптированная под мобильные экраны и собранная под Android.

### Системные требования:
*   Установленные Android SDK, NDK и JDK.
*   Переменная окружения `ANDROID_HOME`, указывающая на ваш SDK.

### Настройка подключения по локальной сети:
1. Узнайте локальный IP-адрес вашего компьютера (например, `192.168.1.183`).
2. Разрешите входящие подключения на порту `8000` в брандмауэре вашего ПК:
   ```bash
   sudo ufw allow 8000/tcp
   ```
3. Подключите телефон к той же Wi-Fi сети, что и компьютер.
4. В приложении на телефоне зайдите в **Settings** и укажите адрес сервера:
   `http://<IP_вашего_компьютера>:8000` (например, `http://192.168.1.183:8000`).

### Запуск в режиме разработки (на подключенном телефоне):
```bash
cd admin-panel-android
npm run tauri android dev
```

### Сборка `.apk` пакета:
```bash
npm run tauri android build
```

## 📝 Разработка и безопасность

*   **База данных:** SQLite работает в WAL-режиме, что позволяет боту и API-серверу работать с ней одновременно без взаимных блокировок.
*   **Офлайн-режим (Queue):** Панели снабжены системой `IndexedDB Queue`. Если соединение с сервером пропадет, ваши действия (баны, изменения настроек) сохранятся локально и синхронизируются автоматически при восстановлении сети.
*   **Гит-безопасность:** Локальные токены, база данных и кэш-файлы сборщиков уже прописаны в корневом `.gitignore` и никогда не попадут в открытый доступ.
